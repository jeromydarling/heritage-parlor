# Heritage Parlor — Full Project Specification

## Overview

Heritage Parlor is a web application featuring 502 Victorian-era parlor games, magic tricks, puzzles, card games, folk games, and word games curated from thirteen public domain books (1857–1917). The project is built in two phases:

1. **Phase 1 (Claude Code)**: Static front-end with all game content, SVG illustrations, printable game sheets, community scaffolding, and Fabrica commission system — deployed as HTML/CSS/JS.
2. **Phase 2 (Lovable / Supabase)**: Full-stack migration with authentication, database-backed features (Game Log, community ratings, seasonal challenges), commerce integrations (Lulu, Stripe Connect), and admin dashboard.

This spec covers both phases. Phase 1 assets are already complete and included in this repository.

### Data Model — entries.json

Each of the 502 entries has the following schema:

```json
{
  "title": "Fox and Geese",
  "source_book": "Cassell's Book of In-door Amusements...",
  "source_author": "Various",
  "source_year": 1881,
  "source_url": "https://www.gutenberg.org/ebooks/49137",
  "category": "board-game",
  "subcategory": "strategy-game",
  "tags": ["strategy", "asymmetric", "capture"],
  "players": "2",
  "difficulty": "intermediate",
  "equipment_needed": ["board", "pieces"],
  "original_description": "Victorian-era description from source...",
  "modern_explanation": "Modern plain-English how-to-play...",
  "fun_fact": "Historical context or trivia...",
  "image_prompt": "Recraft API prompt (used for optional AI illustrations)",
  "family_friendly": true,
  "id": "fox-and-geese",
  "entry_number": 42,
  "playability": "craftable",
  "playability_note": "Board can be drawn on paper...",
  "illustration": "has_original",
  "illustration_note": "...",
  "play_duration": "15–30 min"
}
```

**Categories** (9): `parlor-game`, `card-game`, `magic-trick`, `puzzle`, `word-game`, `physical-game`, `board-game`, `folk-game`, `scientific-recreation`

**Playability ratings** (6): `playable_now`, `easy_to_source`, `craftable`, `specialty_needed`, `extinct_equipment`, `dangerous`

### Data Model — template_map.json

Maps each entry to an SVG template type for diagram generation:

**Template types** (11): `nxn_grid` (59), `morris` (8), `star_polygon` (31), `circle` (30), `chessboard` (14), `graph` (2), `dissection` (8), `card_layout` (49), `magic_sequence` (78), `rules_only` (148), `prop_checklist` (75)

Each entry includes type-specific `params` for the SVG generator (e.g., grid size, number of points, star arms).

### SVG System

Three types of SVGs per game (1,506 total):

1. **Thumbnails** (`svgs/thumbnails/{id}.svg`): Used in the browsing grid. No text overlays. For diagram-type games (board, puzzle), shows the actual game board. For non-diagram games (parlor, word, physical), shows an illustrated scene generated by `svg_illustrations.py`.

2. **Page 1 — Play Sheet** (`svgs/page1/{id}.svg`): Printable page with game title and the playable board/diagram or illustrated scene. Designed for 8.5×11" printing.

3. **Page 2 — Rules Sheet** (`svgs/page2/{id}.svg`): Printable page with game title and full instructions/rules text. Designed for 8.5×11" printing.

**Key design decision**: Every game is printable. Families should never be tied to a phone — they can print game sheets and take them to restaurants, on road trips, etc.

### SVG Generation Pipeline

- `templates.py`: 7 diagram template functions (NxN grid, Morris board, star polygon, circle arrangement, chessboard, dissection, card layout) plus fallback. Uses `template_map.json` parameters.
- `svg_illustrations.py`: 30+ scene-based illustration functions keyed by subcategory. Generates unique illustrated SVGs for non-diagram games (acting games get a stage scene, guessing games get a thought-bubble scene, card games get a card fan, etc.). ~1,160 lines.
- `batch_generate_svgs.py`: Orchestrates batch generation of all 1,506 SVGs.

### Current UI Features

- **Browse grid**: 502 game cards with SVG thumbnails, category filter pills, playability filter pills, search bar, lazy-load pagination (24 at a time)
- **Game detail modal**: Title, category badge, playability badge, play duration, player count, difficulty, original description, modern explanation, fun fact, source attribution, equipment list, print button (generates two-page PDF from SVGs), build guide button (for 36 craftable games), Fabrica commission button (for 36 games with artisan specs)
- **About page**: Project description, source books list, how to use
- **Game Nights page**: 6 themed game night menus (Classic Victorian Evening, Family Fun Night, Dinner Party Brain-Teasers, Rainy Day Games, Road Trip & Restaurant, Holiday Party) with group size tips and curated game lists
- **Submit a Game page**: Form for community game submissions (currently client-side only — no backend)
- **User Suggestions**: Every game detail has a "Got a suggestion?" section for tips/variants (currently client-side only)
- **Dark mode**: Full theme toggle with system preference detection
- **Responsive**: Mobile-first, works on all screen sizes

The commission modal in `app.js` has two modes controlled by `FABRICA_CONFIG.stripeEnabled`:
- **Stripe Connect mode** (Phase 2): POSTs to Fabrica API, redirects to Stripe Checkout
- **Email fallback** (Phase 1): Opens mailto: with pre-filled commission details

36 games have corresponding DIY build guides cross-linked to the Heritage Skills project:

### Database Schema

#### `profiles`
```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  display_name TEXT NOT NULL,
  avatar_url TEXT,
  bio TEXT,
  favorite_categories TEXT[] DEFAULT '{}',
  joined_at TIMESTAMPTZ DEFAULT NOW(),
  is_admin BOOLEAN DEFAULT FALSE,
  contribution_count INT DEFAULT 0,
  games_logged INT DEFAULT 0
);
```

#### `game_log`
```sql
CREATE TABLE game_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  game_id TEXT NOT NULL,
  played_at TIMESTAMPTZ DEFAULT NOW(),
  rating INT CHECK (rating BETWEEN 1 AND 5),
  notes TEXT,
  player_count INT,
  duration_minutes INT,
  would_play_again BOOLEAN
);
CREATE INDEX idx_game_log_user ON game_log(user_id);
CREATE INDEX idx_game_log_game ON game_log(game_id);
```

#### `community_ratings`

### Authentication & User Profiles

**Supabase Auth** with three providers:
- Email/password (with magic link option)
- Google OAuth
- Apple Sign-In

**Profile features**:
- Display name, avatar, bio
- Favorite categories (used by the game picker)
- Game Log stats (games played, hours logged, favorite category)
- Contribution count (suggestions submitted, games submitted)
- Earned badges from seasonal challenges
- Public profile page showing play history and badges

**RLS (Row Level Security)**:
- Users can read all public data (games, ratings, challenges)
- Users can only write their own data (log entries, favorites, suggestions)
- Admin users can read/write all data
- Commission and order data visible only to the owning user + admins

### Feature: Game Log

A personal play journal for tracking games played.

**UI**:
- "Log This Game" button on every game detail page (requires login)
- Quick-log form: date (default today), rating (1–5 stars), player count, duration, notes, "would play again" toggle
- "My Game Log" page in user profile: chronological list with filters by date range, category, rating
- Stats dashboard: total games played, total hours, most-played game, favorite category, play streak

**Backend**:
- `game_log` table with full CRUD
- Aggregate queries for stats (GROUP BY game_id, category, date)

### Feature: Tonight's Game Picker

A smart randomizer to help families pick what to play.

**UI**:
- Prominent "Pick a Game" button on the homepage
- Filter panel: player count (slider), max duration, category checkboxes, difficulty, playability
- Single-game mode: spin wheel animation → reveals one game with detail card
- "Plan a Full Evening" mode: selects 3–4 games of different types that fit within a time budget
- "Not this one" re-roll button
- If logged in: factors in games not yet played and user's favorite categories

**Logic**:
- Weighted random selection: prioritize `playable_now` and `easy_to_source` games
- Evening planner: mix one active/physical game, one brain game, one social/parlor game
- Exclude recently played (from Game Log) unless user opts in

### Feature: Seasonal Challenges

Monthly themed challenges with progress tracking.

**UI**:
- "Challenges" section on homepage (when active challenge exists)
- Challenge card: title, description, progress bar (X of Y games completed), time remaining
- Challenge detail page: list of required games with checkmark for each completed
- Badge display on profile when challenge completed

**Admin-created** via admin dashboard:
- Set title, description, theme, game list, start/end dates, badge SVG
- Monthly cadence (e.g., "October Spooky Parlor Games", "December Holiday Game Marathon")

### Feature: Community Ratings & Tips

**UI**:
- On each game detail: thumbs up/down vote + optional one-line tip
- "Family Tested" badge appears when a game has ≥10 upvotes and ≥70% positive ratio
- Tips section below game description showing approved community tips
- Sort games by community rating in browse grid

**Moderation**:
- Tips go through `pending` → `approved`/`rejected` pipeline
- Admin can bulk-approve or reject from dashboard

### Feature: Printable Game Kits (Lulu Integration)

Curated bundles of printable games shipped as saddle-stitch booklets.

**Pre-built kits**:
- Rainy Day Kit (10 indoor parlor games)
- Restaurant Kit (8 no-equipment games for waiting)
- Road Trip Kit (10 word/guessing games)
- Holiday Party Kit (12 group parlor games)
- Brain Teaser Kit (10 puzzles)
- Custom Kit (user picks any games)

**Flow**:
1. User selects a kit or builds custom selection
2. System generates PDF booklet from SVG page1/page2 sheets
3. POST to Lulu Print API: create print job (saddle-stitch, letter size, 4–48 pages)
4. Lulu handles printing + shipping
5. Order status tracked in `lulu_orders` table

**Lulu API integration**:
- REST API (free, no monthly fee)
- Endpoint: `POST /print-jobs` with interior PDF + cover PDF
- Webhook for status updates (pending → in_production → shipped → completed)
- User pays at checkout, Lulu charges Heritage Parlor wholesale, margin kept as revenue

### Feature: Fabrica Commissions (Stripe Connect)

Handcrafted artisan game boards commissioned through the site.

**Flow**:
1. User clicks "Commission This Game" on any of the 36 eligible games
2. Modal shows materials, dimensions, finish details, price range
3. User fills in name, email, customization notes
4. System creates Stripe Checkout session via Fabrica API
5. User pays deposit through Stripe
6. Artisan receives order notification
7. Order tracked through commission pipeline: pending → paid → in_progress → shipped → completed

**Stripe Connect setup** (built on Lovable.app):
- Heritage Parlor is the platform
- Artisans are connected accounts
- Platform takes a percentage (configurable in admin)
- `FABRICA_CONFIG` in `app.js` already has the toggle and API URL scaffold

### Feature: Community Contributions

**Game Suggestions** (per-game):
- Tip, variant, correction, or house rule
- Moderation pipeline (pending → approved/rejected)
- Approved suggestions shown on game detail page

**Submit a Game**:
- Full form: title, description, source, category, player count, equipment
- Goes to moderation queue
- If approved, admin can publish directly to the entries list

**Leaderboard**:
- "Community Contributors" section showing top contributors
- Points for: approved suggestions (1 pt), submitted games (5 pts), games logged (1 pt)
- Monthly and all-time rankings

### Feature: Game Night Events

**UI**:
- "Game Nights" tab in navigation
- Browse public game night events
- Create a game night: pick a theme, select games, set date/location/max attendees
- RSVP system (going/maybe/declined)
- Host can share a unique link for invitations

**Pre-built themed menus** already exist in Phase 1:
- Classic Victorian Evening
- Family Fun Night
- Dinner Party Brain-Teasers
- Rainy Day Games
- Road Trip & Restaurant
- Holiday Party

### Admin Dashboard

Authenticated admin-only section (`is_admin = true` on profile).

#### Content Management
- Edit game entries (all fields)
- Approve/reject submitted games
- Publish new games
- Manage seasonal challenges (create, edit, activate/deactivate)
- Upload new SVGs or regenerate from templates

#### Moderation
- Suggestion queue: approve/reject with bulk actions
- Submitted games queue
- Reported content (future)
- User management (view profiles, ban if needed)

#### Commerce Tracking
- Fabrica commissions: pipeline view (pending → paid → in_progress → shipped → completed)
- Lulu orders: status tracking with Lulu API sync
- Revenue dashboard: daily/weekly/monthly totals, breakdown by commission vs. print

#### Analytics
- Active users (DAU/WAU/MAU)
- Most popular games (views, prints, logs)
- Community engagement (suggestions, submissions, ratings)
- Game picker usage
- Challenge participation rates
- Revenue metrics

## API Endpoints Needed (Phase 2)

### Auth
- `POST /auth/signup` — Supabase built-in
- `POST /auth/login` — Supabase built-in
- `GET /auth/user` — Supabase built-in

### Game Log
- `POST /api/game-log` — Log a game
- `GET /api/game-log` — Get user's log (paginated)
- `GET /api/game-log/stats` — Aggregated stats
- `DELETE /api/game-log/:id` — Remove log entry

### Favorites
- `POST /api/favorites/:gameId` — Add favorite
- `DELETE /api/favorites/:gameId` — Remove favorite
- `GET /api/favorites` — List user's favorites

### Community
- `POST /api/ratings/:gameId` — Vote + tip
- `GET /api/ratings/:gameId` — Get ratings for a game
- `POST /api/suggestions/:gameId` — Submit suggestion
- `POST /api/games/submit` — Submit a new game
- `GET /api/leaderboard` — Top contributors

### Challenges
- `GET /api/challenges/active` — Current challenge
- `POST /api/challenges/:id/complete/:gameId` — Mark game done
- `GET /api/challenges/:id/progress` — User's progress

### Commerce
- `POST /api/commissions` — Create commission (→ Stripe Checkout)
- `GET /api/commissions` — User's commissions
- `POST /api/lulu/order` — Create print order
- `GET /api/lulu/order/:id` — Order status

### Game Picker
- `GET /api/picker?players=4&maxDuration=30&categories=parlor-game,word-game` — Random game
- `GET /api/picker/evening?players=4&totalMinutes=120` — Evening plan

### Admin
- `GET /api/admin/moderation/suggestions` — Pending suggestions
- `PATCH /api/admin/moderation/suggestions/:id` — Approve/reject
- `GET /api/admin/moderation/games` — Pending game submissions
- `GET /api/admin/analytics` — Dashboard data
- `GET /api/admin/commerce` — Revenue data
